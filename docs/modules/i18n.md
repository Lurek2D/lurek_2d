# I18N

- The `i18n` module is a Feature Systems tier component dedicated to internationalization and localization, enabling games to dynamically adapt to multiple languages and regional formats.

At the core of the module is the `Catalog`, a high-performance, locale-indexed translation store. It loads locale data directly from flat files—typically TOML or JSON—flattening nested structures into dot-separated string keys. To ensure a seamless user experience, the module supports hierarchical fallback chains; if a translation is missing in the active locale, the `Catalog` automatically searches through a defined sequence of fallback languages before defaulting to the raw key.

Beyond simple string replacement, the module features a robust template interpolation engine. It seamlessly substitutes `{name}` placeholders within translated strings with runtime values provided by Lua scripts, securely handling escape sequences. The module also natively supports complex pluralization, implementing locale-specific CLDR plural rules (zero, one, two, few, many, other) with specialized logic for English and Slavic language families. This ensures grammatical correctness when displaying dynamic quantities.

The `i18n` module also manages regional data formatting. It implements locale-aware number formatting, correctly applying regional decimal and thousands separators, alongside sophisticated date formatting that converts Unix timestamps into localized patterns. For development and QA, the module includes a comprehensive coverage analyzer (`coverage_gaps`) that highlights untranslated keys across locale pairs, ensuring localization completeness. The entire suite of translation, formatting, and analysis tools is accessible to scripts via the `lurek.i18n.*` namespace.

## Functions

### `lurek.i18n.buildIndex`

Builds a word-to-keys search index from the catalog.

```lua
-- signature
lurek.i18n.buildIndex()
```

**Returns**

| Type | Description |
|------|-------------|
| `table` | Map table from normalized words to arrays of translation keys. |

**Example**

```lua
do
    lurek.i18n.loadTable("en", { greetings = { hello = "Hello traveler" } })
    lurek.i18n.setLanguage("en")
    local indexed = lurek.i18n.buildIndex()
    print("index built = " .. tostring(type(indexed) == "table"))
    print("has greeting bucket = " .. tostring(indexed.greeting ~= nil or indexed.hello ~= nil))
end
```

### `lurek.i18n.searchIndexed`

```lua
-- signature
lurek.i18n.searchIndexed(index, query, limit)
```

**Parameters**

| Name | Type | Description |
|------|------|-------------|
| `index` | `table` | Index table returned by `buildIndex`. |

| `limit?` | `number` | Maximum key count; zero or nil means no truncation. |

**Returns**

| Type | Description |
