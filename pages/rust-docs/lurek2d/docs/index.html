Exposes the runtime documentation subsystem module. Aggregate documentation infrastructure: catalog, entry models, export, reporting, and schema.

  • Re-export primary types so callers can import from the top-level docs module.
  • Support the doc generation pipeline and IDE tooling data flow.

Modules§

catalog
Expose catalog storage and query operations for documentation entries. Provide in-memory catalog storage for documentation entries collected from Rust source.
entry
Expose normalized entry types for parameters, returns, and metadata. Define normalized documentation record types for lurek API symbols.
export
Expose JSON export builders for completion, hover, and signature payloads. Build JSON payloads for IDE completion, hover, and signature help from doc entries.
report
Expose quality and validation report models for doc coverage analysis. Compute per-entry quality scores from completeness of description, params, and metadata.
schema
Re-export shared schema model types used by documentation tooling. Re-export schema validation types from the lurek_schema crate.
