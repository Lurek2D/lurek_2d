Module input

Expand description

Exposes the input subsystem module. Keyboard, mouse, gamepad, and touch input state aggregation.

  • Event constants for Lua callbacks (keypressed, mousemoved, etc.).
  • Combo gesture detection and input recording for replays.

Modules§

combo
Combo gesture detection and multi-step input sequences. Multi-step key-press combo detection with per-step and total-sequence timeouts.
events
Input event name constants for Lua callbacks. Input event types emitted by the winit event loop and queued for Lua consumption.
gamepad
Gamepad device state, axis/button mapping, and vibration requests via gilrs. Per-slot gamepad state tracking: buttons, axes, connection lifecycle, and per-frame delta sets.
keyboard
Keyboard scan-code state and winit key translation. Per-frame keyboard state machine tracking logical keys, physical scan-codes, and frame deltas.
mouse
Mouse position, button state, cursor kind, and cursor handle management. Per-frame mouse state tracking: position, button held/pressed/released deltas, and scroll accumulators.
recorder
Input event recorder for replays and automated testing. Record and replay input sessions as sparse frame sequences.
touch
Touch-point state tracking for multi-touch surfaces. Multi-touch contact tracking with per-frame pressed/released deltas.
