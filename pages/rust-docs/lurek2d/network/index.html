Exposes the networking subsystem module. Multiplayer networking: TCP, WebSocket, and relay transports with binary message framing.

  • Host/client model with lobby state machine, peer management, and game-state sync.
  • Background async runtime (Tokio) for non-blocking socket I/O and HTTP helpers.

Modules§

constants
Shared numeric limits and protocol constants used across all network layers. Numeric limits for peer connections, channels, and buffer sizes.
error
NetworkError type covering socket, protocol, and framing failures. Unified error enum for all network subsystem failures.
host
Host-side peer management: accept loop, peer registry, and disconnect handling. ENet host wrapper owning a non-blocking UDP socket and all peer slots for one endpoint.
http
Blocking HTTP GET/POST helpers used for matchmaking and asset fetching. Synchronous HTTP client built on ureq for GET, POST, PUT, PATCH, DELETE, HEAD, and OPTIONS.
lobby
Lobby state machine: room creation, join, leave, and member list tracking. LAN lobby discovery via timed UDP broadcast listen on a fixed port.
message
NetMessage enum and binary framing used on every transport. Wire-format value type (NetValue) mirroring Lua’s dynamic type system for cross-peer messaging.
net_sync
Game-state snapshot diffing and reliable sync packets sent between peers. Entity snapshot capture and wire serialization for networked state.
net_thread
Background Tokio thread that owns the async socket runtime; started at engine init. Background network thread that owns all blocking I/O (HTTP, TCP, WebSocket).
relay
Relay server client: punch-through, forwarding, and relay session lifecycle. Relay ticket encoding and decoding for room+peer identification over the wire.
tcp
Raw TCP transport: connect, send, receive, and graceful close. Non-blocking TCP connection pool for the background network thread.
websocket
WebSocket transport wrapping tungstenite; mirrors the TCP interface. Manage a pool of active WebSocket connections keyed by caller-assigned ID.
