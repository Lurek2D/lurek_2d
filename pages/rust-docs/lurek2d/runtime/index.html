Module runtime 

Source
Expand description

Exposes the core runtime subsystem module. Engine runtime foundations: configuration, shared state, and error types.

  • Loads conf.toml into a typed Config struct consumed by all subsystems.
  • Provides SharedState for mutable cross-module communication during a frame.
  • Defines EngineError variants and slot-map resource keys.

Engine-wide error types and snapshot helpers. Defines EngineError — the engine-wide error enum covering all subsystem failures.

No-window Lua runtime used by headless startup mode. Implements the no-window headless runtime path for script automation and CI use.

Log message identifiers and log-level override helpers. Stable, structured log message identifiers for all engine subsystems.

Message catalog loader and lookup API. Embedded TOML-based message catalog for runtime log and display text.

Runtime mode parsing and display helpers. Defines RuntimeMode enum with four variants: gui, tui, headless, and cli.

OS-level utilities including clipboard, system info, and platform detection. Operating system detection utilities for platform-specific code paths.

Slot-map key types used by runtime-owned resources. Typed slotmap keys for every engine resource pool (textures, fonts, sounds, particles, etc.).
shared_state
Shared mutable runtime state consumed by app and Lua callbacks. Central mutable state container shared across all engine subsystems during a frame.
